git指南：常用指令手册

Git 是一个开源的分布式版本控制系统，可以有效、高速地处理从很小到非常大的项目版本管理。Git 是基于 Linux内核开发的版本控制工具，与常用的版本控制工具不同，它采用了分布式版本库的方式，不必服务器端软件支持，使源代码的发布和交流极其方便，Git 最为出色的是它的合并跟踪能力。

一、Git 安装过程

1、Windows 下安装

git下载地址：https://git-scm.com/downloads

下载安装包后，一路下一步即可。

2、Cent os 下安装

安装指令：sudo yum install git -y

等待下载完毕后，输入git --version，查看 git 版本。

3、Ubuntu 下安装

安装指令：sudo apt-get install git -y

等待下载完毕后，输入git --version，查看 git 版本。

二、配置本地仓库

1、 初始化 Git 仓库

进入文件夹内，输入 git init，然后输入 la查看是不是有 .git目录，如果有就成功了。

2、配置 name 和 email

配置名字：git config user.name "your name"
配置邮箱：git config user.email "your email"
查看配置：git config -l
删除名字：git config --unset user.name
删除邮箱：git config --unset user.email

四、本地文件上传到指定仓库

git init: 这个命令会将当前目录初始化为一个 Git 仓库，从而使其成为 Git 可以管理的仓库。

git add README.md: 这个命令将名为 README.md 的文件添加到 Git 的暂存区，准备将其提交到本地仓库。

git add .: 这个命令将当前目录下所有未被 Git 追踪的文件添加到暂存区，这是一个批量添加操作。

git commit -m "commit description": 这个命令会将暂存区的所有更改提交到本地仓库，并附带一条提交描述信息，以便日后更容易理解该提交的内容。

git remote add origin git https://github.com/xxx/xxx.git: 这个命令会将你的本地仓库与一个名为 “origin” 的远程仓库关联起来。”origin” 是默认的远程仓库名称，通常用于指代主要的远程仓库。

git push -u origin main: 这个命令会将本地仓库的内容推送到远程仓库，并将本地的 main 分支与远程仓库的 main 分支进行关联。 -u 选项用于将本地分支与远程分支关联起来，这样以后可以使用 git push 和 git pull 来更容易地同步代码，而无需每次都指定远程分支和本地分支。
