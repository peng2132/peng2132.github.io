剑指offer：数组中重复的数字

一、题目描述

1、题目

剑指offer：数组中重复的数字

在一个长度为 n 的数组里的所有数字都在 $0$ 到 $n-1$ 的范围内。数组中某些数字是重复的，但不知道有几个数字重复，也不知道每个数字重复了几次。请找出数组中任意一个重复的数字。例如，如果输入长度为7的数组$[2,3,1,0,2,5,3]$，那么对应的输出是2或者3。存在不合法的输入的话输出-1。

数据范围：$0≤n≤10000$
进阶：时间复杂度$O(n)$ ，空间复杂度$O(n)$

2、示例

输入：[2,3,1,0,2,5,3]
返回值：2
说明：2或3都是对的

二、题目分析

题目的意思非常简单，一句话总结就是：一个特定范围内的数组中找到任意一个重复的数字。

1、双重for循环

通过循环来比较数组中的每一对元素来查找重复。

时间复杂度是$O(n^2)$：因为有两个嵌套循环，每个循环都遍历整个数组。
空间复杂度是 $O(1)$：没有使用除了输入数组之外的额外空间。

2、for-each 循环

使用一个额外的数组来记录每个数字出现的次数。一旦某个数字的出现次数超过1，就立即返回该数字。在时间上更高效，但牺牲了一些空间来达到这个目的。

时间复杂度：$O(n)$。只有一个循环，遍历一次数组。
空间复杂度：$O(n)$。使用了一个与输入数组同等大小的额外数组来记录每个数字出现的次数。

3、set集合

创建一个 HashSet，在循环内部，首先检查当前元素 i 是否已经在 set 中：

  • 如果 set 已包含 i，则返回 i 作为第一个重复的元素。
  • 如果 set 不包含 i，则将 i 添加到 set 中，然后继续循环。

时间复杂度是 $O(n)$，空间复杂度是$O(n)$：使用了一个额外的 HashSet 来存储元素。在最坏情况下（没有重复元素），这个集合将包含所有 n 个元素。
