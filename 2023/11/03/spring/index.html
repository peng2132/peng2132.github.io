Spring框架：Spring MVC

一、MVC的定义

MVC 是 Model — View — Controller 的简称，它是一种架构模式，它分离了表现与交互。它把软件分为三个核心部件：模型、视图、控制器。

  • Model（模型）：是程序的主体部分，主要包含业务数据和业务逻辑。在模型层，还会涉及到用户发布的服务，在服务中会根据不同的业务需求，更新业务模型中的数据。

  • View（视图）：是程序呈现给用户的部分，是用户和程序交互的接口，用户会根据具体的业务需求，在 View 视图层输入自己特定的业务数据，并通过界面的事件交互，将对应的输入参数提交给后台控制器进行处理。

  • Controller（控制器）：Controller 是用来处理用户输入数据，以及更新业务模型的部分。控制器中接收了用户与界面交互时传递过来的数据，并根据数据业务逻辑来执行服务的调用和更新业务模型的数据和状态。

Spring在实现MVC时，也结合⾃⾝项⽬的特点，做了⼀些改变，可以配合下图理解：

服务员就像是控制器（Controller），她接收你的请求，知道应该去哪里获取所需的资源来满足这个请求。服务员不会亲自去厨房做菜，而是将你的订单告诉厨师。模型（Model）在这里是厨师和他们使用的食材。他们代表了业务逻辑和数据处理，知道如何根据服务员提供的信息（也就是你的订单）来准备一道菜，随后将这道菜给服务员，服务员会把菜也就是视图（View）放到你面前。

我们作为客户，不需要知道菜是如何做的，所以对于我们而言，最重要的其实就是三件事：

  1. 去餐厅：建立连接
  2. 找服务员点餐：发起请求
  3. 拿到点的饭：得到响应

路由映射: 当⽤⼾访问⼀个 URL 时, 将用户的请求对应到程序中某个类的某个⽅法的过程就叫路由映射。

@RequestMapping是Spring MVC中常见的注解，主要用于设定接口的路由映射。比如，当服务器接收到路径为/say的请求时，它会调用say方法来处理这个请求。

@RequestMapping即可修饰类，也可以修饰方法，当修饰类和方法时，访问的地址是类路径+方法路径。

@RequestMapping标识一个类：设置映射请求的请求路径的初始信息

@RequestMapping标识一个方法：设置映射请求请求路径的具体信息

三、请求

访问不同的路径, 就是发送不同的请求。

在发送请求时, 可能会带⼀些参数。所以Spring的请求，主要就是如何传递参数到后端，以及后端如何接收。

1、传递单个参数

使用浏览器进行传参：http://127.0.0.1:8080/user/say?name=张三

Spring MVC 会根据⽅法的参数名, 找到对应的参数, 赋值给⽅法。

2、传递多个参数

和接收单个参数⼀样，直接使⽤⽅法的参数接收即可，使⽤多个形参。

3、后端参数重命名

使用@RequestParam注解来实现这种重命名。这个注解允许我们在处理请求参数时，将请求参数的名称（在HTTP请求中的名称）映射到我们在代码中定义的变量名。@RequestParam(value = "name",required = false) String username，HTTP请求中的参数"name"被映射到了方法参数username。

4、传递对象

当一个方法需要处理大量参数时，我们的方法声明就会包含很多形参。而且，每次我们需要添加新的参数时，都必须修改方法声明。为了让代码更简洁，我们可以将这些参数封装成一个对象。Spring MVC 也可以⾃动实现对象参数的赋值，⽐如 Person 对象：

四、响应

Http响应结果可以是数据，也可以是静态⻚⾯，也可以针对响应设置状态码，Header信息等。

1、@Controller

如果写的是@RestController返回的其实是几行字。

@RestController是一个Spring MVC的注解，等同于@Controller和@ResponseBody的组合。它标记一个类作为控制器，并且这个控制器的方法返回的数据会直接写入HTTP响应的正文，而不是用来解析视图。如果希望方法返回的是视图（例如HTML页面）而不是数据，可以只使用@Controller，这样返回的字符串会被视为视图名，Spring MVC会用这个名字去查找并渲染对应的视图模板。

2、@ResponseBody

@ResponseBody是一个在Spring MVC中使用的注解，既可以用在类上，也可以用在方法上。当它用在类上时，表示该类中的所有方法返回的都是数据，而不是视图。当它用在方法上时，表示该方法返回的是数据。也就是说，给类添加@ResponseBody注解等同于给该类中的所有方法都添加了@ResponseBody注解。

@RestController注解也有类似的效果。当一个类被@RestController注解时，这等同于给该类添加了@Controller和@ResponseBody注解，这意味着该类中的所有方法返回的都是数据。如果一个类中的方法既有返回数据的，又有返回视图的，那么可以只在需要返回数据的方法上添加@ResponseBody注解，这样这些方法返回的就是数据，而没有添加@ResponseBody注解的方法返回的就是视图。

加上 @ResponseBody 注解, 该⽅法就会把 “/index.html” 当做⼀个数据返回给前端。
